# 早安毛健康 · 品牌語氣規範（單一事實來源）

> **適用範圍**：所有早安毛健康內容產出 SKILL — 新文撰寫、改稿、小文、推薦文、大小改自動化。
> 寫新文章時遵守這些規則；改既有文章時用這份檔案判斷哪些段落「不像早安毛」需要改。
> 來源：整合 `brand.md`、`tone-and-manner.md` 及改稿實務經驗的完整版。

## 1. 一句話定位

像「養 3 隻毛孩、有做功課的資深飼主朋友」在跟讀者聊天。專業守住，但不擺教科書架子。第一人稱固定用「我們」（不用「小編 / 我 / 本站」）。

## 2. 語氣判準（只記一條）

> 這句話放在部落格文章裡讀起來自然嗎？太口語或太書面都改掉。

唸出來如果覺得「太隨便」（例：「精神和胃口也會一起掉」）或「太正式」（例：「認知功能呈現顯著衰退」）→ 往中間調。論文腔、翻譯腔、AI 排比句通過不了這關。

## 3. 必改 — 一看到就標問題

| 訊號 | 範例（壞） | 改寫方向 |
|------|-----------|---------|
| 第一人稱漂移 | 小編爬文 / 我發現 / 本站 | 改為「我們」 |
| 論文式開場 | 「現代科學研究發現…」「綜觀整體而言…」 | 平實切重點 |
| 標題黨 | 震驚！99% 飼主不知道 | 拿掉 |
| 過度可愛 | 寶貝毛毛超級無敵可愛的啦～ | 拿掉 |
| 說教口吻 | 飼主應當嚴格遵守… | 改為「建議 / 可以…」 |
| 空泛轉場 | 接下來要介紹… / 讓我們一起來了解吧 | 直接進主題 |
| 翻譯腔 | 配色邏輯 / 關鍵差異 / 辨識度極高 | 配色 / 最大的差異 / 很好認 |
| 預設刻板印象開頭 | 很多人以為… / 你有沒有發現… | 直接平實切入 |
| 「不是 A 才是 B / 不是 A 而是 B」過用 | （連續多句使用） | 全篇 ≤ 2 處，超過拿掉 |
| 偏書面口語替代 | 單純（→ 簡單）/ 你家毛孩（→ 毛孩） | 換口語版 |
| 把人歸成「型」 | 安安靜靜待你身邊的陪伴型 | 平實描述「平時就安靜地陪伴在你身邊」 |
| 硬塞俏皮話 | 根本是貓界的話癆擔當 / 完全不同風格的室友 | 比喻只在真能幫助理解時用 |
| AI 造詞 / 外語直譯 | 什麼來頭 / 隨便養 / 聰明過人 / 值得一提的是 | 用台灣飼主實際會講的話改寫（見 § 3.1） |
| 書面四字成語硬用 | 莫過於 / 堪稱 / 可謂 / 不容小覷 / 首屈一指 / 與生俱來 | 拆成白話短句，或直接刪掉 |
| 論文式轉場詞 | 綜上所述 / 總而言之 / 換言之 / 簡而言之 / 由此可見 / 顯而易見 | 用「所以」「也就是說」或直接省略 |

### 3.1 AI 造詞 / 外語直譯 — 常見壞詞 → 自然改法對照

以下詞彙在中文母語者的寵物文章中幾乎不會出現，多為 AI 生成或英文直譯。一看到就標問題並改寫：

| 壞詞 | 為什麼不自然 | 自然改法 |
|------|-------------|---------|
| 什麼來頭 | 「來頭」用在人身上帶負面意味，用在狗品種很怪 | 品種背景 / 怎麼來的 / 是什麼品種 |
| 隨便養 | 台灣人不會把「隨便」當形容詞修飾「養」 | 好照顧 / 省心 / 不用太費心 |
| 聰明過人 | 「過人」是形容人超越其他人，用在狗身上語意矛盾 | 超聰明 / 聰明到不行 / 智商很高 |
| 值得一提的是 | 論文 / 導覽手冊口吻，真人聊天不會這樣說 | 比較特別的是 / 有趣的是 / 另外 |
| 莫過於 | 書面文言，口語幾乎不用 | 最…的就是 / 說到…第一個想到 |
| 堪稱 / 可謂 | 新聞稿用語 | 算是 / 根本就是 / 簡直是 |
| 不容小覷 | 翻譯腔 | 不能忽略 / 要特別注意 |
| 無疑是 | 論文腔 | 絕對是 / 就是 |
| 與生俱來 | 書面四字成語 | 天生就有 / 本來就… |
| 息息相關 / 密不可分 | AI 愛用的空泛連接 | 有很大的關係 / 直接影響 |
| 至關重要 / 舉足輕重 | 翻譯腔 | 很重要 / 一定要注意 |

**判斷原則**：唸出來如果像在讀報告或聽 AI 語音助理 → 改掉。參考 kuanchiang.com.tw 等真人寵物文章的用字：具體、口語、帶生活感，例如「智商就是……堪用」「想說帶牠去散步」「結果沒想到」。

## 4. 寫作 13 條通則

### 基礎七條

1. **形容詞晚於事實**：硬資訊先出、修飾後出。
2. **主張要有理由**：每個「適合 / 不適合」都能追到機制。
3. **動作 / 狀態 > 抽象名詞**：能寫畫面就不要名詞化。
4. **每句扛新資訊**：刪一句要缺一個事實，同義反覆直接砍。
5. **主語回到對象本身**：少用「這種感覺 / 此情況」當主語；破折號插入易漂移 → 改「逗號 + 補主詞 + 完整子句」。
6. **碰到專有名詞追一層**：英文 / 年代 / 人名 / 地名都追「為什麼、是誰、怎麼來」。
7. **不替讀者決定感受**：不用「令人心疼 / 超療癒」這類引導詞。

### 服務具體讀者三條

8. **在地化脈絡**：氣候寫濕熱、住家寫公寓、價格用台幣、平台舉 PTT/Dcard/Threads。
9. **借用飼主圈共同語言**：貓皮狗骨、跟屁蟲、玻璃胃、狗系貓、貓主子、毛爸媽、跟屁蟲、傻白甜… 第一次出現用「」當錨點。
10. **對策給到生活層級**：不停在「建議建立安全感」這種抽象，寫成「出門前 10 分鐘不要互動」。

### 呈現與作者聲音三條

11. **多重觀察點用 1234 編號子標**：H3 切開，每項一段，方便手機 scan。
12. **第一人稱用「我們」**：不用「小編 / 我 / 本站」。
13. **多維度框架選擇**：同一事實有多個真實角度，選對讀者有用的那個。例：認知能力「介於一般貓與工作犬之間」→ 寫給準飼主用「學習能力強、像 2-4 歲幼童」。

### 段落與錨點三條

14. **每個 H2 ≥ 1 處顯式比較**：「跟什麼比？」一句比較 = 兩個資訊。「布偶貓底毛比波斯貓薄」 > 「布偶貓的毛不容易打結」。
15. **每個 H3 ≥ 1 個錨點事實**：讀者會複述給朋友的具體事實。寫完段落自問「讀者會記住哪一句？」答不出來 → 加 / 砍。
16. **避免假精準**：主觀類事實（智商、認知、遺傳率）若用無來源的窄百分比 → 改寬區間或質性描述。客觀可測（體重、價格、年份）才能精準。

## 4-2. E-E-A-T Experience 經驗感寫法（改文 HTML 產出時遵循）

Google 的 E-E-A-T 第一個 E 是 Experience（經驗）。真人寫的寵物文章之所以讀起來有溫度，關鍵是「經驗感」——讀者能感受到作者真的養過、遇過、處理過。以下是從台灣真人寵物部落格（kuanchiang.com.tw 等）歸納出的經驗感寫法技巧，早安毛的「我們」人格套用時，把「小編」替換成「我們」即可。

### 4-2-1. 五種經驗感句型模板

**① 事實 + 自家驗證**
先陳述一個通用事實，緊接自家寵物的經驗做佐證。
- 模板：「[事實陳述]，我們家的[貓/狗]也是這樣，[具體細節]。」
- 範例：「比熊的淚痕跟飲食真的有關，我們家那隻只要吃到雞肉底的飼料，眼下就會紅一圈。」

**② 期待 vs 現實（驚喜包敘事）**
寫出「原本以為 → 結果其實」的落差，帶來真實感。
- 模板：「[當初的預期]，結果沒想到[出乎意料的結果]。」
- 範例：「當初覺得邊牧很聰明應該很好教，結果沒想到牠聰明到會反過來訓練你。」

**③ 內心獨白 + 具體金額或時間**
用「想說」「覺得」帶出當下心境，再附上具體數字增加可信度。
- 模板：「那時候想說[當時的想法]，[具體金額/時間/地點]，[後來的結果]。」
- 範例：「那時候想說帶去洗牙好了，掛號加麻醉血檢一次就噴了 8,000 多，才知道貓的牙科真的不便宜。」

**④ 生活場景類比（讀者換位）**
用人類日常情境類比寵物行為，讓不養寵物的人也秒懂。
- 模板：「你[人類日常情境]嗎？[寵物做的事]其實就像是這樣。」
- 範例：「你會跟一個才剛認識的人馬上勾肩搭背嗎？狗狗也一樣，牠需要時間確認你沒有敵意。」

**⑤ 坦承不確定 / 自嘲**
刻意暴露不完美，增加可信度（AI 不會自嘲）。
- 模板：「老實說[承認不足的事]，不過[轉折到有用資訊]。」
- 範例：「老實說我們也試過好幾種除淚痕的方法，有的根本沒用，目前覺得最有效的是先換飼料再搭配每天擦。」

### 4-2-2. 經驗感插入頻率

- 每個 H2 段落至少出現 1 處經驗感句型（上述 ①～⑤ 任一種）
- 經驗感句型不集中在同一個 H2，要分散在全文
- FAQ 段落不強制，但如果能用 ① 或 ⑤ 帶出答案更好

### 4-2-3. 經驗感 vs AI 感的快速判斷

| 經驗感（✅ 留） | AI 感（❌ 改） |
|---------------|-------------|
| 有具體動物、具體情境、具體數字 | 空泛概述、沒有任何個案 |
| 有時間軸（「剛帶回家的時候」「養了三年後」） | 無時間感的靜態描述 |
| 有情緒詞但自然（「嚇到」「傻眼」「心疼」） | 書面情緒詞（「令人動容」「不禁感嘆」） |
| 承認失敗或不確定 | 每件事都給出完美答案 |
| 用「我們」帶出觀察 | 用「研究顯示」「專家指出」但沒附來源 |

### 4-2-4. 注意事項

- 早安毛的「我們」= 團隊口吻，不要寫成「我」或「小編」
- 經驗可以是「我們觀察到」「我們遇過」「我們家的貓」「之前有飼主跟我們分享」——不一定要是自己養的，也可以是讀者 / 飼主社群回饋
- 經驗不能捏造具體品牌或醫院名（除非真有）。可以模糊化：「有飼主在 PTT 分享」「社團裡常看到有人問」
- 經驗段落仍然要符合 YMYL 原則：涉及醫療的經驗分享後面要接「但還是建議帶去給獸醫看」

### 4-2-5. AI 撰文時的經驗感限制（2026-06-24 新增）

上述模板 ①～⑤ 是為**人類編輯**設計的——他們能從真實飼養經驗中取材。**AI 撰文時不得使用這些模板生成虛構第一人稱敘事**，因為 AI 沒有真實飼養經驗，套用模板必然產生捏造內容，讀者和編輯一眼能看出來。

**AI 全面禁止的句型**：
- ❌「我們第一次…結果沒想到…」（模板 ②）
- ❌「那時候想說…結果一查…」（模板 ③）
- ❌「老實說我們看到…也嚇了一跳」（模板 ⑤）
- ❌「我們原本以為…結果沒想到…」（模板 ②）
- ❌「你有沒有…的經驗？X 就是把這件事做到極致。」（模板 ④）

**AI 應改用的經驗感替代方式**：
- 第三方歸因：「有飼主分享」「社團裡常看到有人問」「常見情況是」
- 品牌建議句（保留「我們」）：「我們建議放客廳靠牆」「我們建議先從虎皮入門」
- 具體數據自帶說服力，不需要假裝「我們親身驗證過」

## 5. H2 / H3 格式

- H2：口語化、可帶問句／驚嘆號；**不用**國字標號（一、二、三）；不加粗。
- H3：**✦ 符號**或數字；不加粗；用讀者視角口語命名（飼主在 PTT 怎麼問？而非教科書怎麼分類）。
- FAQ H3 加「Q：」前綴，**不加 ✦**（避免重複）。

## 6. FAQ

- 每題 H3 加「Q：」，回答**不加**「A：」前綴。
- 每題 40-120 字，完整封閉式回答。
- 比正文更口語。
- H2 標題：「核心關鍵字 + 常見問題」。

## 7. 段落節奏與結構

### 基本規則

- 一段一重點，≤ 4 行（80-120 字）
- 短句（8-15 字）混長句（20-35 字），不連續 3 句同長度
- 每段標 1 處核心資訊粗體（有資訊量的句子片段，非單一名詞）
- 不連續 2 段相同方式開頭
- 混合：陳述句、反問句、數據開頭、案例開頭
- 反問句每 300-500 字可用 1 次
- **bullet 拿掉後段落仍站得住 → 拿掉 bullet**（避免目錄感）
- **H2 視覺換行硬規則**：每個非 FAQ／非 CTA 的 H2 段落至少出現 1 個 `<ul>` / `<ol>` / `<table>`，或至少 2 個 `<strong>`。整段連續敘事超過 3 個 `<p>` 沒有視覺換行 → 標問題

### 句型多樣性

- 「不是…而是…」「不是…才是…」等公式化對比句型，全篇合計 ≤ 2 處
- 避免連續 3 段都用「結論 + 解釋 + 補充」三段式句構；每個 H2 裡應至少 1 段換成「提問 + 回答」或「對比 + 收束」結構
- **魚頭魚尾句式對比**：H2 開場的「魚頭結論句」與該 H2 段末的「魚尾收束句」句長應差距 ≥ 10 字，避免開頭與結尾同長度級距造成的平板感

### 語句節奏與轉折詞池

- **轉折詞池輪替**：「不過、然而、只不過、此外、同時、其實、另一方面、相反地」。同一 H2 內，同一轉折詞不得連用 2 次以上；整篇的前 3 名轉折詞不得佔全部轉折詞用量 > 60%
- **功能詞不連用（則／也／又／還）**：對比／遞進功能字同一句內 ≤ 1 次；同一段內 ≤ 2 次
  - ❌「身體則呈現淺色調，眼睛則是清透的藍色。」（同句雙「則」）
  - ✅「身體呈現淺色調，眼睛則是清透的藍色。」
- **口語助詞節制使用**：「唷 / 呢 / 了 / 喔」等句尾助詞每 400–600 字至多出現 1 次；連續 2 段都用同一個助詞 → 標問題
- **避免 AI 感排比**：三件事並列時，前後兩件用完整句、中間一件改用短片語，或把第三件抽出單獨成句。四件以上並列改 `<ul>` 列點

### 段落拆分時機

- **轉折前拆段**：出現「不過」「但」「然而」等對比轉折時，考慮在轉折前拆段
- **功能不同就拆**：同一段在講兩件不同面向的事 → 拆成兩段
- **100+ 字 + 雙重功能**：一段超過 100 字且涵蓋「觀察描述」和「建議行動」→ 拆開
- ❌ 不要為了拆而拆：兩句話緊密相關且加起來 < 80 字，不必硬拆
- **獨立收束句**：一段事實描述結束後，把結論或影響拉出來單獨成一行（≤ 30 字），用「所以」「因此」「這也是」開頭
- **因果鏈不切斷**：多個事實構成因果鏈（A→B→C→D），寧可段落長一些也不要切斷

### 資訊排列順序

- **數據在前、描述在後**：先用數據或統計建立可信度，再展開具體症狀或表現
- **結論在前、條件在後**：先給讀者答案（正不正常、要不要就醫），再補充判斷依據

### 書面與白話交織（不工整性原則）

真人寫文章不會整篇維持同一種語感。解釋機制、引用數據時可以用書面語（「急劇下降」「優先攻擊」「經由胎盤傳給胎兒」），講到讀者切身影響或行動建議時拉回白話（「更容易感染其他疾病」「就可能把病毒一起帶回家」）。同一段裡書面和白話交替出現 = 正常；整段都是書面語或整段都是口語 = 標問題。

### 刪除多餘的預告句與重述句

以下類型的句子在大改分析時看到就標問題：

1. **預告型句子**——接下來的內容已經在解釋，不需要先預告。❌「但背後的原因完全不同。幼犬是因為…」→ ✅ 直接寫「幼犬是因為…」
2. **重述型句子**——前面已經傳達過同樣的資訊，後面又覆述。
3. **AI 式貼心提醒句**——「這樣你就能知道什麼時候該緊張了」「記住這幾點就夠了」→ 直接刪掉。
4. **H3 問句不在正文重複**——H3 標題已是問句時，正文直接給答案，不要第一句再問一遍。

## 8. 標點符號規範

### 速查 5 條

- 並列描述同類事物的不同項目（每項是完整複句）→ **分號**，不要句號斷開。
- 列舉並列詞語 → 頓號，最後一項用「和／及」（最後一項前不加頓號）。
- 善用連接詞減少句號（「四隻腳掌是白色的，像穿了小手套，而且下巴通常也是白的」）。
- 結論段獨立，用「所以」帶出，不黏在描述段尾巴。
- 破折號插入 → 改「逗號 + 補主詞 + 完整子句」。

### 完整規範

- **逗號 vs 頓號**：逗號隔開複句內的分句；頓號只用在並列詞語之間。三項以上並列詞語用頓號，最後一項前改用「和／及／或」，不再加頓號
- **分號**：用於並列的複句之間或分項列舉（「首先…；其次…；最後…」）。❌ 不用於轉折／因果關係
- **破折號「——」**：兩個全形破折號為一組，全篇 ≤ 2 處。短段落（< 150 字）避免使用，優先改為逗號 + 補主詞
- **省略號**：固定 6 點「……」占兩格。❌ 不用「...」「。。。」「⋯」
- **嘆號**：全篇節制使用，建議 ≤ 5 個。❌ 禁止「？！」「！？」並用
- **引號**：雙引號「」用於引述、強調、專有名詞；單引號『』只用於引號內再引
- **數字區間**：中文正文用全形波浪號 `～`（✅ `18～20 小時` ❌ `18-20 小時`）
- **全形 vs 半形**：中文句內一律全形標點；中文與英文／數字之間加半形空格（盤古之白）

### 口語感與連接詞

- 善用連接詞（而且、而、也、同時、另外）串句，減少句號硬停頓
- 用逗號 + 連接詞串句：兩句話主題相同只是角度不同（補做法、補原因、補替代方案），用逗號 + 連接詞比句號更自然
- 「所以」多層使用：不只段落間收束，段落內部也能帶因果
- 「也」做跨 H3 銜接：編號 H3 之間可在下一個 H3 開頭用「也」串接前一個 H3 的語脈
- 「只要…就…」條件句型：表達因果時優先考慮此結構，比單純陳述更有畫面

## 9. 引用來源具名規則

- **台灣本土寵物品牌 / 競品部落格不具名**（汪喵星球、毛起來、窩窩、HaloPet、毛孩時代）→ 用「據統計 / 據研究」。
- **國外來源可具名**：AKC、PetMD、ASPCA、VCA Hospitals、Cornell、Merck Veterinary Manual 等。
- 大改建議要求補來源時，優先指向國外權威或台灣官方（農業部動保、各縣市動保處）。

### 9.1 引用句式多樣化（禁止連續「根據XXX」）

全篇引用來源時，**禁止連續使用同一種句式**。「根據XXX」全篇最多出現 2 次（保留給學術研究引用），其餘必須交替使用以下句式：

| 句式類型 | 範例 | 適合情境 |
|---------|------|---------|
| 句中嵌入（來源＋顯示/發現） | 「AKC 的品種資料顯示，比熊犬平均壽命約 14 至 15 年。」 | 強調來源權威性 |
| 結論先行，來源放括號 | 「比熊犬的膝蓋骨異位盛行率約 12%（Nilsson et al., 2018）。」 | 數據密集段落，讓閱讀更流暢 |
| 動詞變化（指出/提醒/提到） | 「VCA 動物醫院的資料指出…」「BFCA 特別提醒…」「PetMD 特別提到…」 | 建議類、警告類資訊 |
| 以…來看 | 「以 AKC 公布的品種標準來看，肩高約 24 至 30 公分。」 | 描述型資訊 |
| 超連結本身承載（最簡潔） | 「比熊犬的[平均壽命約 14 至 15 年](URL)，在小型犬中算長壽。」 | FAQ、行文極簡處 |

**禁用**：「這個數字來自 XXX」（讀起來像翻譯腔）。

**自審檢查**：完稿後 Ctrl+F 搜尋「根據」，超過 2 次就改寫。同一種句式連續出現 2 次以上也要打散。

## 13. 豐富性原則（內容密度判準）

### 具體量化數字下限

任何涉及「該多久一次／該多少量／該多少錢」的段落，必須給具體數字或區間：

| ✅ 正確寫法 | ❌ 常見抽象寫法 |
|---|---|
| 每週梳毛 2–3 次，換毛季增至每日一次 | 定期梳毛即可 |
| 成貓每日 3–4 餐、定時定量 | 定時餵食、避免暴食 |
| 寵物級 2.5–7 萬、繁育級 4–15 萬、賽級 7–30 萬 | 價格 3 萬到數十萬不等 |
| 每日互動遊戲 15–30 分鐘 | 每天陪牠玩 |

大改分析時若看到「定期」「適量」「一段時間」等空話 → 標「缺具體量化」，指令中要求補數字或區間。

### 價格段落分級規則

凡是「XX 價格」「XX 多少錢」這類段落，不得只給單一數字。必須給至少 3 個分級或場景（血統／等級、通路、產品定位）。

### 具名資源 > 抽象描述

外部資源（領養平台、政府網站、社群、認證機構、醫療機構等）必須用真實名稱並帶超連結，不得以通用描述含混：

| ✅ 正確 | ❌ 含混 |
|---|---|
| 「MeetPets 台灣認養地圖」 | 「民間送養平台」 |
| 「全國動物收容系統」 | 「各地公立收容所」 |
| 「TICA、CFA 國際貓協會認證血統書」 | 「國際血統認證」 |

大改分析時看到抽象描述 → 標「具名資源不足」，指令中要求改為具體名稱 + 連結。

## 14. 關鍵字語句通順原則

關鍵字塞入正文時，語句通順度和標點符號正確性優先於原始關鍵字形式。搜尋引擎能理解語意相近的變體，但讀者讀到硬塞的斷句會覺得不自然。

- 「英短和美短的外觀差異其實很好辨認。」（加「和」）
- 「英短、美短都是台灣常見品種貓。」（加頓號）
- 「比較英短與美短的個性前，先了解牠們的背景。」（加「與」）

唯一例外：H1、meta description 維持關鍵字原始形式。正文中第一次出現也建議保留原始形式，之後一律以語句通順為優先。

## 15. 魚骨寫作法與金字塔法（改文 HTML 產出時遵循）

### 魚骨寫作法

每個段落都像一條魚：
- **魚頭（主題句）**：第一句就給結論或核心觀點，讀者掃讀時只看這句就能抓到重點
- **魚骨＋魚肉（補充）**：用細節、數據、案例支撐主題句
- **魚尾（收尾）**：重複強化核心觀點，加深讀者印象

大改分析時若段落第一句不是破題 → 標「缺主題句（魚頭）」。

### 金字塔法

- 先講結論，再用補充強化結論
- H2 若有過渡段，則是該章節的金字塔頂端；沒過渡段時，第一個 H3 或第一段 bullet 承擔頂端結論
- 過渡段選用（非強制）。若寫：80-150 字、第一句給結論、不用「接下來介紹…」

## 16. 題型語氣調整

不同文章類型的語氣側重不同，大改時須根據文章類型判斷語氣是否適當：

- **品種／比較文**：先幫讀者分辨差異，再補性格、照顧與適合族群
- **醫療／風險文**：語氣更直接，先講風險等級與下一步，不要寫得太繞
- **飼養／教學文**：多用情境句和步驟句，像在陪讀者處理眼前問題
- **飲食／推薦文**：先講能不能、值不值得，再展開條件、限制與選法
- **生活知識文**：保留輕鬆口氣，但核心資訊仍要先講

共通底線不變：先說人話、先給判斷、每段只做一件事。
