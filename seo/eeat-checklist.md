# E-E-A-T 與資訊正確性檢核 · 早安毛健康

> 用途：分析既有文章時，逐條對照判斷有無 E-E-A-T 缺口。寵物健康屬 YMYL，標準比一般生活內容嚴格。
> 適用範圍：醫療、營養、行為、品種特性、用藥、急救、繁殖、選購等任何「飼主會根據文章下決定」的內容。
> 與 google-algo-signals.md 分工：本檔聚焦「內容支柱要補什麼」（4 大支柱 + YMYL 紅線）；google-algo-signals.md 聚焦「演算法在抓什麼」（Helpful Content、Scaled Content、Reviews System、Core Update）。同一篇文章常常兩邊都觸發，產出大改建議時建議在指令上標註對應 reference 編號。

## 一、Experience（真實經驗）

> Experience 是 2022 年 12 月 Google 把 E-A-T 升級為 E-E-A-T 時新增的支柱，等於「First-hand experience」。寵物健康內容尤其重要——飼主會直覺辨別「這人到底有沒有養過動物」。

| 檢核項 | 判準 | 缺口時的處理 |
|--------|------|------------|
| 第一人稱集體經驗 | 文章中至少出現 1-2 次「我們爬文發現…」「我們自己養過…」「我們整理了…」 | 建議在 H2 開場或結語段補 |
| 飼主真實情境 | 有沒有具體場景描述（搬家、出門上班、季節轉換、半夜異常）而非抽象描述 | 建議補「PTT/Dcard 飼主常反應…」「許多飼主在 XX 情境下會發現…」 |
| 觀察級的具象細節 | 描述行為 / 症狀時是否寫到「具體可見的動作」而非抽象形容 | 建議改寫成可觀察的動作 |
| 飼主圈共同語彙 | 有沒有用上 ≥ 2 個飼主圈俗稱（玻璃胃、跟屁蟲、開罐頭…） | 建議補；參考 brand-voice § 11 |
| 觀察 / 比較 / 對照 | 有沒有「我們對比了 / 我們爬了 N 篇 / 我們訪談了…」這類「實際做了什麼功課」的揭露 | 建議在文章開頭或某 H2 補一段 |

**Experience 缺口的常見徵兆**：通篇都像「百科全書」，沒有「有人養過」的氣味，讀起來跟維基百科或寵物店官網很像。

**與 Helpful Content 的關聯**：Google 的 Helpful Content 自評題其中一條是「Does the content provide original information, reporting, research, or analysis?」 → Experience 不足會直接觸發 Helpful Content 反訊號。詳見 google-algo-signals.md § 一.2、§ 二。

## 二、Expertise（專業度）

| 檢核項 | 判準 | 缺口時的處理 |
|--------|------|------------|
| 專有名詞中英對照 | 出現英文 / 拉丁文 / 醫學名詞時有「中文（英文縮寫）」格式（例：肥厚性心肌病 HCM） | 建議補對照 |
| 機制解釋 | 對「為什麼會 / 為什麼不」的問題有給機制（不是只說「容易得 XX」） | 建議補機制（例：「腸胃敏感是因為遺傳上腸道菌相結構…」） |
| 數據單位明確 | 數字後面有單位（公克、毫升、新台幣、年、週） | 建議補單位 |
| 不過度斷言 | 醫療建議是否避免「絕對」「一定」「永遠」等 | 建議改為「通常 / 多半 / 一般而言」 |
| 專家審稿暗示 | 是否在文末或開頭暗示有獸醫 / 專業人員審過（如「本文經獸醫師審閱」） | 對醫療類文章建議補；非醫療類不強制 |
| 鑑別診斷意識 | 對症狀類內容，是否提示「這也可能是 XX」而非單一斷定 | 建議補可能的其他原因 |

**Expertise 缺口的常見徵兆**：把多個原因混為一談；只列症狀不講機制；把「相關」當成「因果」；數據沒單位；醫療建議下得太重。

## 三、Authoritativeness（權威性）

| 檢核項 | 判準 | 缺口時的處理 |
|--------|------|------------|
| 引用來源具體 | 引用文獻 / 研究時有「機構名稱 + 年份」最低限度 | 建議補來源 |
| 國外權威優先 | 引用是否來自 AKC / PetMD / ASPCA / VCA / Cornell / Merck Veterinary Manual / RSPCA / WSAVA 等 | 缺乏時建議補 1-2 個 |
| 台灣本土依據 | 提到台灣現況時是否引用農業部動保、台灣獸醫師公會、各縣市動保處 | 缺乏時建議補 |
| 不具名引用台灣競品 | 汪喵星球、毛起來、窩窩、HaloPet、毛孩時代 → **不可具名**（用「據統計」「據研究」） | 有具名 → 標問題，要求改 |
| 連結可用 | 文章內外連結是否仍有效 | 失效 → 標問題 |

**Authoritativeness 缺口的常見徵兆**：引用「研究指出」但沒說哪個研究；引用「專家表示」但沒說哪位專家；數據沒來源；只引用台灣競品而沒有國外權威。

## 四、Trustworthiness（可信度）

| 檢核項 | 判準 | 缺口時的處理 |
|--------|------|------------|
| 風險與缺點誠實 | 介紹品種 / 食物 / 用品時有提缺點與風險（不是只講優點） | 建議補風險段 |
| 假精準偵測 | 主觀類事實（智商、遺傳率、情感力）有沒有用無來源的精準百分比？ | 改寬區間或質性 |
| 廣告意圖透明 | 文末有沒有偽裝成「客觀建議」的硬推銷？ | 建議改為延伸閱讀 / 軟性引導 |
| 矛盾資訊一致性 | 同篇文章前後對同一事實的描述是否一致？ | 不一致 → 標問題，要求統一 |
| 時間敏感資訊更新 | 是否有「2020 年研究…」這類過舊資訊但講當前現況？ | 建議補近年資料 |
| 醫療免責提示 | 醫療類文章是否提示「以上為一般資訊，請以獸醫診斷為準」？ | 醫療類建議補 |

**Trustworthiness 缺口的常見徵兆**：只報喜不報憂；無來源精準數字；硬推商品；資訊矛盾；資料明顯過舊。

## 五、寵物健康類專屬紅線

以下任何一條出現 → 必標為「優先修改」：

1. **下醫療診斷**：「你的狗有 XX 病」「這就是 XX」這類確診式語言 → 改為「可能是 XX，建議就醫」。
2. **建議停藥 / 換藥 / 自行投藥**：未經獸醫指示的用藥建議 → 全部刪除，改為「請與獸醫討論」。
3. **餵食人類食物的安全性錯誤**：常見錯誤如「狗可以吃巧克力」「貓可以喝牛奶」（其實有些可以、有些不行，要寫對） → 標為「資訊正確性需查證」。
4. **混淆貓狗用藥 / 劑量**：貓不能用犬類除蚤藥（如含 permethrin） → 標為「致命錯誤可能」。
5. **品種特性 vs 個體差異混淆**：把「品種傾向」寫成「個體必然」（例：「布偶貓一定很黏人」） → 改為「布偶貓多半…，但個體差異存在」。
6. **絕育 / 繁殖建議**：未交代風險與時機 → 建議補。
7. **疫苗 / 驅蟲時程**：給出具體時程但沒交代「以獸醫評估為準」 → 補免責。
